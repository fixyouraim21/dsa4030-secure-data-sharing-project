# dsa4030-secure-data-sharing-project
# Group 12 – Secure Data Sharing

## Project Overview
This project simulates two organizations securely exchanging datasets on a daily basis.
It demonstrates encryption, digital signatures, signature verification, integrity checking
(hash comparison), and secure file transfer using open-source tools (GPG, OpenSSL).

## Team & Responsibilities
| Member | Responsibility |
|---|---|
| Member 1 | Environment setup, dataset generation, SQLite database, logging |
| Member 2 | Encryption, digital signatures, signature verification, hash checking, secure transfer simulation |
| Member 3 | Security testing, documentation, screenshots, final report, presentation |

## Dataset
- **File:** `data/employee_dataset.csv`
- **Records:** 100,000 synthetic employee/HR records
- **Fields:** employee_id, first_name, last_name, email, gender, department, job_title, salary, hire_date, employment_status, country, office_location, phone_number
- **Generated by:** `scripts/generate_employees.py` (uses the `faker` library for realistic synthetic data)

To regenerate the dataset:
```powershell
python scripts/generate_employees.py
```

## Database (SQLite)
The database (`db/project.db`) tracks metadata about the dataset file as it moves through
the secure-sharing process — from generation through encryption, signing, and transfer.

**Table: `file_metadata`**
| Column | Description |
|---|---|
| file_id | Auto-incrementing ID |
| filename | Name of the tracked file |
| record_count | Number of records in the file |
| sha256_hash | SHA-256 hash (filled in by Member 2's integrity check) |
| encrypted | Whether the file has been encrypted (Yes/No) |
| signed | Whether the file has been digitally signed (Yes/No) |
| sender_org | Organization sending the file |
| receiver_org | Organization receiving the file |
| transfer_status | Pending / Sent / Verified |
| created_at | Timestamp of record creation |

### Scripts (run in order, from the project root)
```powershell
python scripts/step1_create_db.py          # Creates the database and table
python scripts/step2_insert_metadata.py    # Inserts metadata for employee_dataset.csv
python scripts/step3_view_metadata.py      # Views current contents of the table
python scripts/step4_expand_table.py       # Adds security/transfer tracking columns
```

## Logging
All scripts log their actions to `logs/activity.log` using a shared logging configuration
(`scripts/logger_config.py`). Every script in the project (Member 1, 2, and 3's) imports this
config so all activity — dataset generation, database changes, encryption, signing, verification,
and transfer — is captured in one timestamped audit trail.

To view the log:
```powershell
Get-Content logs\activity.log
```
